# DRAM Decode Tools for Intel Skylake (skx_edac)

This project provides a set of kernel modules and tools to decode physical memory addresses into DRAM components such as socket, memory controller, channel, rank, row, column, bank, and bank group. It is specifically designed for Intel Skylake systems using the `skx_edac` driver.

## Files

### skx_dram_decode.c

This kernel module demonstrates how to access the internal `skx_decode` function from the `skx_edac` driver using a kprobe. It allocates a test page, retrieves its physical address, and decodes it. The decoded information is printed to the kernel log.

### skx_dram_decode_addr.c

This module is similar to `skx_dram_decode.c` but allows the user to specify a physical address as a module parameter. It decodes the given address and prints the DRAM layout information to the kernel log.

### batch_decode.sh

This is a shell script that repeatedly loads and unloads the `skx_dram_decode_addr` module for a set of randomly generated physical addresses. The number of addresses to decode can be specified as an argument. Results are saved to a log file.

### Makefile

This Makefile builds both kernel modules. It is compatible with standard Linux kernel build procedures.

### decode_results.log

This is the output log file generated by `batch_decode.sh`. It contains all decoded results or error messages from each address tested.

## Usage

Ensure the `skx_edac` driver is loaded and compatible with your hardware.
